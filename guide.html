Which graph should I open?

Both views render the same 230 nodes and all their relationships. Pick based on what you want to do.

Explore connections

  • Search by name, CLI syntax, or description

Force-Directed View · graph.html

The 3D spring-layout map

A self-contained HTML file powered by 3d-force-graph (vasturiano / Three.js). Nodes float freely and settle via a physics simulation. Runs entirely in the browser — no server needed.

4

Use filters and search

Narrow the graph with group/tier filters, surface focus buttons, or the search box. See the features section below for details on each.

Force-Directed View · Features

Everything graph.html can do

🔍 Search

Search across all fields

The search box matches against method name, description, metadata, CLI syntax, MCP tool name, and category. Results highlight instantly. An amber notice appears when matched nodes are hidden by active filters.

⬡ Group filters

Group filters

Toggle entire node groups on and off: Surfaces · API (148) · Open Bugs · Fixed Bugs · Patterns · References. Multiple groups combine — filters stack independently.

📶 Tier filters

Tier (surface count) filters

Isolate commands by how many surfaces they're on. Mutually exclusive (radio). Click again to clear.

All 5 · 72 4 surf · 9 3 surf · 54 2 surf · 11 Planned · 2
🎯 Surface Focus

Surface Focus buttons

The header buttons — CLI · MCP · JS · Python · Java — highlight all commands on that surface without hiding any nodes. Mutually exclusive. Click again to clear.

▼ Collapse

Category collapse

Click a category node, then use the ▼ Collapse commands button in the sidebar to hide that category's command nodes. Multiple categories can be collapsed simultaneously. Search notices update accordingly.

↗ Docs links

Direct docs links

Every command node's sidebar includes a ↗ docs link that jumps straight to the corresponding methods/ page on GitHub — covering syntax, known bugs, and workarounds.

📊 Stats panel

Live stats panel

The bottom-right panel updates in real time as you apply filters: visible commands (out of 148), open bugs, fixed bugs, total nodes, and total links currently rendered.

? Guide overlay

Built-in guide

Click ? Guide in the header for a quick-reference overlay covering navigation, filters, tier colors, and surface focus. Close with Esc, the × button, or by clicking outside.

Layered Planes View · graph-layered.html

Six tiers, one scene

An alternative rendering of the same 230-node dataset — powered by Three.js r161 with OrbitControls. Nodes sit on six fixed horizontal planes stacked by tier, making the hierarchy immediately readable without waiting for a physics simulation to settle.

Left-drag to rotate · Scroll to zoom · Right-drag to pan

Click a node

Layer controls

Toggle layer visibility with checkboxes · show/hide labels per layer · drag the spacing slider to fan layers apart or compress them.

Layered View · Features
☰ Layer toggles

Show / hide layers

The left panel has a checkbox for each of the 6 tiers. Uncheck a layer to hide its plane, all its nodes, and all its labels in one click — useful for focusing on commands only or bugs only.

𝐀 Label controls

Per-layer labels

Each layer row has a Labels button. Root, Surfaces, and Categories show labels by default; Commands, Bugs, and Refs are off. Toggle individually or check All Node Labels for every node at once.

↕ Spacing slider

Layer spacing

Drag the Layer Spacing slider (40 – 180 px, default 90) to fan the planes apart for a panoramic view or compress them to compare layers side-by-side. Edges and ring markers update live.

◉ Info panel

Node info panel

Click any node to open the right-side info panel with the same content as the force-directed sidebar: full API syntax per surface, bug tags, severity, and workarounds. Click the node again or × to close.

⊙ Ring markings

L0 – L5 tier rings

Each plane carries a faint ring with a label (e.g. L3 · Commands) at its edge. The rings stay visible during orbit so you always know which tier you're looking at from any angle.

Open it: lana-20.github.io/vibium-wiki/graph-layered.html ↗ — or open graph-layered.html locally.

What's in the docs

The wiki lives alongside the graph in the same repo. All pages are Markdown, cross-linked via [[wiki-link]] notation.

DirectoryContents

guides/ Getting started with JS/TypeScript · MCP setup for Claude Code and Gemini CLI · recording format and player.

Tips

Quick reference

graph.html — Force-Directed

Find a method fast

Type in the search box. Matches highlight immediately. Click the highlighted node for full syntax across all 5 surfaces.

Check surface availability

Use Tier filters to see commands on exactly N surfaces. Use Surface Focus to highlight all commands on one surface.

Investigate a bug

Search the bug ID (e.g. B3). Click the red bug node to see affected commands and workarounds. Follow the ↗ docs link for the full report.

Declutter the graph

Toggle off Fixed Bugs and References to focus on active API methods. Collapse noisy categories from their sidebar button.

Compare surfaces

Click a command node → sidebar shows all 5 surfaces at once. Planned entries show ⬜ planned; missing show —.

Reset everything

Reload the page to reset all filters. Or click ✕ on the sidebar, click active filter buttons again to toggle them off.

graph-layered.html — Layered Planes

Focus on one tier

Uncheck all layers except the one you want. Try Commands-only (L3) to see just the 148 API methods, or Bugs-only (L4) to scan open issues.

Read node labels

Labels are on by default for L0–L2. Turn on All Node Labels to see every node named, or use individual layer Labels buttons.

See the full hierarchy at once

Drag the Layer Spacing slider right to fan tiers apart. Orbit to a top-down view to see all six planes as a stack.

Inspect a node

Click any node to open the info panel on the right — same API syntax and bug data as the force-directed sidebar. Click again or press × to close.

Orient yourself while orbiting

Each plane has an L0–L5 ring label at its edge. If you lose your place, the ring tells you which tier you're looking at from any angle.

Reset the camera

Reload the page to return to the default orbit angle and spacing. All layer toggles and label states reset to defaults.
