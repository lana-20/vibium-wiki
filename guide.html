Knowledge Graph

The 3D interactive map

The graph is a self-contained HTML file powered by 3d-force-graph (vasturiano / Three.js). It runs entirely in the browser — no server needed.

1

Open locally

Double-click graph.html, or from Terminal:

open graph.html        # macOS
# or via GitHub Pages:
# https://lana-20.github.io/vibium-wiki/graph.html
2

Rotate & zoom

Drag to rotate · Scroll to zoom · Right-drag to pan

3

Click a node

Click any node to highlight its neighbors and open the sidebar with full API syntax. Click the same node again to clear.

4

Use filters and search

Narrow the graph with group/tier filters, surface focus buttons, or the search box. See below for details on each.

Features

Everything the graph can do

🔍 Search

Search across all fields

The search box matches against method name, description, metadata, CLI syntax, MCP tool name, and category. Results highlight instantly. An amber notice appears when matched nodes are hidden by active filters.

⬡ Group filters

Group filters

Toggle entire node groups on and off: Surfaces · API (148) · Open Bugs · Fixed Bugs · Patterns · References. Multiple groups combine — filters stack independently.

📶 Tier filters

Tier (surface count) filters

Isolate commands by how many surfaces they're on. Mutually exclusive (radio). Click again to clear.

All 5 · 72 4 surf · 9 3 surf · 54 2 surf · 11 Planned · 2
🎯 Surface Focus

Surface Focus buttons

The header buttons — CLI · MCP · JS · Python · Java — highlight all commands on that surface without hiding any nodes. Mutually exclusive. Click again to clear.

▼ Collapse

Category collapse

Click a category node, then use the ▼ Collapse commands button in the sidebar to hide that category's command nodes from the graph. Multiple categories can be collapsed simultaneously. Search notices update accordingly.

↗ Docs links

Direct docs links

Every command node's sidebar includes a ↗ docs link that jumps straight to the corresponding methods/ page on GitHub — covering syntax, known bugs, and workarounds.

📊 Stats panel

Live stats panel

The bottom-right panel updates in real time as you apply filters: visible commands (out of 148), open bugs, fixed bugs, total nodes, and total links currently rendered in the graph.

? Guide overlay

Built-in guide

Click ? Guide in the header for a quick-reference overlay covering navigation, filters, tier colors, and surface focus. Close with Esc, the × button, or by clicking outside.

Filters

Combining filters

Group filters and tier filters compose independently. Surface Focus overlays on top of whatever is visible. Here are useful combinations:

Example: commands on exactly 3 surfaces
Tier: 3 surf → 54 commands visible (fill, hover, scroll…)
Example: all open bugs for CLI commands only
Surface Focus: CLI + Group: Open Bugs ✓ → CLI commands highlighted, open bugs visible
Example: find a method — does it have a Python equivalent?
Search: "hover" → click node → Sidebar: Python row
Search notice: If you search while group or tier filters are active, an amber notice appears showing how many matching nodes are currently hidden. Toggle filters back on to reveal them.
Wiki

What's in the docs

The wiki lives alongside the graph in the same repo. All pages are Markdown, cross-linked via [[wiki-link]] notation.

bugs/ B1–B33 CLI bugs · MB1–MB10 MCP bugs · JS, Python, Java client bugs. Each entry covers reproduction steps, affected surfaces, and workarounds.

guides/ Getting started with JS/TypeScript · MCP setup for Claude Code and Gemini CLI · recording format and player.

Tips

Quick reference

Find a method fast

Type in the search box. Matches highlight immediately. Click the highlighted node for full syntax across all 5 surfaces.

Check surface availability

Use Tier filters to see commands on exactly N surfaces. Use Surface Focus to highlight all commands on one surface.

Investigate a bug

Search the bug ID (e.g. B3). Click the red bug node to see affected commands and workarounds. Follow the ↗ docs link for the full bug report.

Declutter the graph

Toggle off Fixed Bugs and References to focus on active API methods. Collapse noisy categories from their sidebar button.

Compare surfaces

Click a command node → check the sidebar syntax table for all 5 surfaces at once. Planned entries show ⬜ planned; missing show —.

Reset everything

Reload the page to reset all filters and state. Or click ✕ on the sidebar, click active filter/tier buttons to toggle them off.
