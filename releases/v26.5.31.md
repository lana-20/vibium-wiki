# Release v26.5.31

**Published:** 2026-06-01  
**Author:** @hugs  
**Tag:** https://github.com/VibiumDev/vibium/releases/tag/v26.5.31  
**Previous:** v26.3.18

A broad bug-fix release driven almost entirely by community reports. Headline fix restores the npm package (`dist/` was missing in v26.3.18). Fixes span CLI, MCP, JS/Python/Java clients.

---

## What was fixed

### npm / packaging
- `dist/` folder restored — `require('vibium')`, `import 'vibium/sync'`, TypeScript declarations all work again (#103, #127, #100)
- ESM binary resolution fixed (#62)
- `prepack` hook now rebuilds `dist/` and refuses to publish an incomplete package

### Engine (all clients)
- `fill` on `<textarea>` no longer crashes with "Illegal invocation" (#117 · CLI B7, MCP #155 · MCP)
- `select` now errors on non-matching option (was silent success); now matches by visible label, not `value` attribute (#140 · B5)
- `waitUntil` / `waitForFunction` accept bare JS expressions, not only arrow functions (JS #123, #131, #145)
- `clock.install()` guard — clock methods now error if `install()` wasn't called first (was silent no-op) (JS #125) ⚠️
- Script exceptions surface real error messages (#111)

### MCP server
- `browser_evaluate ""`, absent attributes, text-less elements no longer cause `invalid_union` (MCP #154, MCP #153, MCP #157)
- Stringified numbers accepted (#85)
- `browser_count` no longer crashes on numeric result (MCP #149)
- `browser_storage_state` parses BiDi's structured cookie values (MCP #150)
- `browser_set_cookie` derives domain from current page when omitted (MCP #152)
- `browser_screenshot { annotate: true }` no longer crashes (MCP #156)

### CLI & error messages
- No more Node.js stack traces on ordinary failures — binary message + exit code (#161)
- Errors categorized correctly — validation/not-found failures no longer mislabeled as timeouts. Affects Python/Java: `TimeoutError` no longer raised for non-timeout failures (#64) ⚠️

### Python client
- `page.eval()` alias added (#144)
- `element.bounds()` now supports dict-style access alongside attribute access (#147)
- Large pipe messages (screenshots) no longer crash with `LimitOverrunError` (#110, #168)

### Java client
- Fixed: `waitForURL` (#129), `addScript`/`addStyle` (#130), `dispatchEvent` (#132), `dragTo` (#134), `onError`/`collectErrors` (#136), clock time formats (#137)
- `el.highlight()` implemented engine-side (#133)

---

## Behavior changes (breaking / surprising)

| Change | Impact |
|---|---|
| `select` now errors on non-matching option | Code that relied on silent failure will now see an error |
| Clock methods error if `clock.install()` not called | Silent no-ops become explicit errors |
| Non-timeout failures no longer raise `TimeoutError` | Python/Java error handling may need updates |
