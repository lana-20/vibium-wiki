Vibium Knowledge Graph — Guide

v26.5.31 · ~148 commands · 5 surfaces · 17 categories

What this graph shows

Every Vibium API method mapped across all 5 access surfaces (CLI, MCP, JS, Python, Java), grouped into 17 functional categories. Bugs and design patterns are attached to the commands they affect.

Node types

  • Purple large sphere — Vibium root node (the entry point to the whole graph)
  • Surface nodes (5) — CLI, MCP, JS Client, Python Client, Java Client. Each branches to the categories it covers.
  • Category rings (17) — functional groups: Browser, BrowserContext, Navigation, Finding, Clicking, Keyboard, Mouse, Touch, Frame, Network, Storage, Dialog, Download, Recording, Clock, Evaluate, Wait
  • Command dots (~148) — individual API methods. Their color encodes availability (see Availability Tiers below).
  • Red/orange nodes — open or partial bugs, linked to the commands they affect
  • Green nodes — fixed bugs (historical record)
  • Pink/purple nodes — design patterns (recommended usage sequences)

Availability tiers — command node colors

A command's color tells you how many surfaces it's available on. The tier filter buttons in the header match these colors.

Header controls

  • Group filter row — toggle entire node types on/off: Surfaces, API (all commands), Open Bugs, Fixed Bugs, Patterns, Reference. Hiding "API" removes all command dots; tier filters then have no effect.
  • Tier filter row — isolate commands by availability tier. Click a tier to show only those commands; click again to clear. Works within the "API" group.
  • Surface Focus row — highlight all commands available on a specific surface. The graph dims everything else. Only one surface can be active at a time. Click the active button again to clear, or press Escape.

Group filters and tier filters combine: tier filters refine within the API group; Surface Focus highlights across all visible nodes without hiding anything. Example: "All 5" tier only + CLI Surface Focus = see which cross-platform commands are on the CLI.

Clicking nodes

  • Click any node — highlights its direct connections; opens detail panel in the right sidebar showing CLI/MCP/JS/Python/Java syntax for command nodes
  • Click same node again — clears the highlight
  • Click a surface node — highlights all commands on that surface (same effect as Surface Focus buttons)
  • Click a category node — highlights all commands in that category
  • Click background — closes sidebar and clears highlight
  • Escape key — clears all highlights and closes sidebar

Sidebar — category collapse

Click a category node, then look in the right sidebar. You'll see a ▼ Collapse commands button. Collapsing a category hides its command dots from the graph — useful for de-cluttering large categories like Finding or Clicking. Click ▶ Expand commands in the sidebar to restore them.

Search

The search box highlights nodes by name, description, CLI syntax, MCP tool name, or surface. Results dim non-matching nodes. Clear the box to reset.

Example workflows

  • "What can the CLI do?" → click CLI in Surface Focus — 52 matching commands highlighted
  • "What's Java-only (no CLI/MCP)?" → disable all tier buttons except Client, then click Java in Surface Focus
  • "Explore the Recording API" → click the Recording category node; sidebar shows 6 commands; use collapse/expand to toggle them
  • "Find all open bugs affecting navigation" → enable Open Bugs group, search navigation
  • "Compare clock syntax across surfaces" → search clock, click a clock command to see CLI/MCP/JS/Python/Java in the sidebar
  • "Show only universally available commands" → deactivate all tier buttons except All 5
  • "De-clutter the graph" → collapse large categories (Finding, Clicking, Browser) via the sidebar button after clicking each
